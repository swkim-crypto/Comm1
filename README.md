# Vibe Coding · C/S 프롬프트 공유 도구 (다중 강의 + 인원 설정)

강사 1명 ↔ 수강생 N명이 같은 서버로 프롬프트·파일을 주고받는 Client/Server 도구입니다.

## 새로워진 점 (v2)
- **강의별 방 분리** — URL 경로 끝에 강의명을 붙이면 완전히 분리된 시스템이 됩니다. 여러 강의가 동시에 살아 있습니다.
- **강사가 인원 설정** — 강사가 처음 입장하면 수강생 수를 입력하고, 그 수만큼 자리가 생깁니다. (이후 늘리기 가능 / 줄이기는 제출물 보호를 위해 불가, 상한 40명)
- LAN/Render 모두에서 **복사·전체 복사** 동작.

## 실행
```bash
npm install
node server.js     # 기본 포트 3000
```

## 접속 (강의명을 경로 끝에)
| 역할 | 주소 예시 |
|------|-----------|
| 강사 | `https://<도메인>/lao2024?host=1` |
| 학생 | `https://<도메인>/lao2024?number=1` |
| 안내 | `https://<도메인>/lao2024` (번호 선택) |

- 강의가 바뀌면 `lao2024` 자리만 다른 이름으로 (영문소문자·숫자·`_`·`-`). 예: `dx0626`, `nonsu-canal`.
- 같은 이름으로 들어오면 같은 방을 공유합니다. 강의마다 고유한 이름을 쓰세요.

## 흐름
1. 강사가 `…/<강의명>?host=1` 접속 → **수강생 수 입력 → 수업 시작**.
2. 학생에게 `…/<강의명>?number=1`, `…?number=2` … 식으로 번호별 주소 배포.
3. 학생: 이름 입력 → 프롬프트·파일 제출 / 강사 배포 자료 받기.
4. 강사: 그리드에서 학생 클릭 → 제출 프롬프트 목록·파일 확인, **이 프롬프트 복사 / 전체 복사**.
5. **전체 초기화**는 현재 강의(방)만 비웁니다 (인원 설정은 유지, 다른 강의 영향 없음).

## 저장 위치
- 학생 파일: `uploads/<강의명>/<번호>_<이름>/`
- 강사 자료: `materials/<강의명>/`
- 상태: `data/state.json`

## ⚠️ Render 무료 플랜 주의 (영속성)
무료 플랜은 **재배포·약 15분 미사용 슬립 시 임시 디스크가 초기화**됩니다 → 방·제출물이 사라질 수 있습니다.
- 강의 중에는 정상 동작, 끝나고 시간이 지나면 초기화될 수 있음.
- "당분간 유지"가 꼭 필요해지면: Render **Disk(유료)** 추가 후 환경변수 **`DATA_DIR`**(예: `/var/data`)만 설정하면 영속화됩니다. (코드는 이미 `DATA_DIR` 지원)

## Render 설정
- Build Command: `npm install`
- Start Command: `node server.js`
- 필수 환경변수: 없음 (`PORT`는 Render 자동). 영속화 시에만 `DATA_DIR` 추가.
