# Vibe Coding · C/S 프롬프트 공유 도구 (다중 강의 + 인원 설정 + 1:1 답변)

강사 1명 ↔ 수강생 N명이 같은 서버로 프롬프트·파일을 주고받는 Client/Server 도구입니다.

## 주요 기능
- **강의별 방 분리** — URL 경로 끝에 강의명을 붙이면 완전히 분리된 시스템. 여러 강의 동시 운영.
- **강사가 인원 설정** — 강사 첫 입장 시 수강생 수 입력(이후 늘리기 가능 / 줄이기 불가, 상한 40명).
- **전체 배포** — 강사가 프롬프트·파일을 모든 학생에게 배포.
- **1:1 답변(신규)** — 상세 보기 창에서 특정 학생의 프롬프트를 골라 **답변 보내기** → 그 학생 화면에만 "강사 답변"으로 표시. 프롬프트를 선택하면 "내 프롬프트 #n 답변"으로 연결되어 보이고, 답변 삭제도 가능.
- **복사 / 전체 복사** — LAN(http)·Render(https) 모두에서 동작.

## 실행
```bash
npm install
node server.js     # 기본 포트 3000
```

## 접속 (강의명을 경로 끝에)
| 역할 | 주소 예시 |
|------|-----------|
| 강사 | `https://<도메인>/lao2024?host=1` |
| 학생 | `https://<도메인>/lao2024?number=1` |
| 안내 | `https://<도메인>/lao2024` (번호 선택) |

강의가 바뀌면 `lao2024` 자리만 다른 이름으로 (영문소문자·숫자·`_`·`-`). 같은 이름이면 같은 방 공유.

## 흐름
1. 강사 `…/<강의명>?host=1` → **수강생 수 입력 → 수업 시작**.
2. 학생에게 `…/<강의명>?number=1`, `?number=2` … 번호별 주소 배포(QR 권장).
3. 학생: 이름 입력 → 프롬프트·파일 제출 / 배포 자료 받기 / **강사 답변 확인**.
4. 강사: 그리드에서 학생 클릭 → 프롬프트 선택 후 **답변 보내기**(1:1), 또는 **전체 복사 / 이 프롬프트 복사**.
5. **전체 초기화**는 현재 강의(방)만 비움(인원 설정 유지, 다른 강의 영향 없음).

## 저장 위치
- 학생 파일: `uploads/<강의명>/<번호>_<이름>/`
- 강사 자료: `materials/<강의명>/`
- 상태(프롬프트·답변 포함): `data/state.json`

## ⚠️ Render 무료 플랜 주의 (영속성)
무료 플랜은 **재배포·약 15분 미사용 슬립 시 임시 디스크가 초기화**됩니다 → 방·제출물·답변이 사라질 수 있습니다.
필요해지면 Render **Disk(유료)** + 환경변수 **`DATA_DIR`**(예: `/var/data`)만 설정하면 영속화됩니다(코드는 이미 지원).

## Render 설정
- Build Command: `npm install`
- Start Command: `node server.js`
- 필수 환경변수: 없음(`PORT` 자동). 영속화 시에만 `DATA_DIR` 추가.
