# Vibe Coding · C/S 단계 — 1:20 프롬프트/자료 공유 도구

강사 1명 ↔ 수강생 20명이 같은 서버를 통해 프롬프트와 파일을 주고받는 Client/Server 데모입니다.
사다리 C/S 데모와 동일하게 **강사 노트북에서 서버를 띄우고, 학생들은 같은 WiFi로 접속**합니다.

## 실행 방법

```bash
npm install      # 처음 한 번만 (express, multer 설치)
node server.js   # 서버 시작 (기본 포트 3000)
```

서버를 켜면 콘솔에 접속 주소가 출력됩니다.

## 접속 주소

| 역할 | 주소 | 설명 |
|------|------|------|
| **강사** | `http://localhost:3000/?host=1` | 좌(1) 자료 배포 / 우(3) 수강생 5×4 그리드 |
| **학생** | `http://<내IP>:3000/?number=1` | 끝의 `number`를 1~20으로 (학생마다 다르게) |

내 IP는 터미널에서 `ipconfig`(Windows) / `ifconfig`(Mac) 로 확인 → 예: `192.168.0.23`.
학생에게는 `http://192.168.0.23:3000/?number=1`, `...?number=2` … 식으로 번호별 주소를 나눠주세요.

## 동작 요약

- **강사**: 좌측에서 프롬프트(텍스트)·파일을 배포 → 학생 화면에서 받아갈 수 있음.
  우측 5×4 버튼에 이름을 직접 입력해 둘 수도 있고(미리 명단), 학생이 입장하며 입력해도 됨.
  버튼의 **상세 보기**를 누르면 그 학생의 제출 프롬프트가 **순서대로 리스트박스**로 뜸 + 파일 다운로드.
- **학생**: `?number=N`으로 입장 → 이름 입력 → 프롬프트·파일 제출. 강사 배포 자료 받기.
- **파일 저장**: 학생이 올린 파일은 서버의 `uploads/1_이름/` … `uploads/20_이름/` 폴더에 저장됨.
- **동기화**: 1초 폴링 — 20명 모두 같은 서버 상태를 봄. *"자료의 진실은 서버가 가진다."*
- **초기화**: 강사 화면 우상단 **전체 초기화** → 모든 이름·제출물 + 서버 저장 폴더 내용까지 삭제.

## 폴더 구조

```
prompt-cs/
├── server.js          # Express 서버 (API + 파일 저장)
├── public/index.html  # 강사·학생 화면 (역할은 URL로 분기)
├── package.json
├── data/state.json    # 상태 저장(자동 생성, 재시작해도 유지)
├── uploads/           # 학생 제출 파일 (1_이름/ …, 자동 생성)
└── materials/         # 강사 배포 파일 (자동 생성)
```

> 참고: 파일을 강사 노트북의 실제 폴더에 저장하므로 로컬 실행용입니다.
> (Render 같은 임시 파일시스템에 올리면 재배포 시 업로드 파일이 사라집니다.)
